# Development Policies

These policies guide AI agent behavior for this project.

## Policy Files
- `docker-first.md`: Docker-based development protocol
- `autonomous-git.md`: Git commit/push automation
- `ml-project-qc.md`: ML project quality control process (5 phases)
- `tool-timeout.md`: Timeout handling and fallback strategies

## Usage
When starting work on this repo, agent should:
1. Read all .policies/ files
2. Follow protocols defined within
3. Update policies if patterns emerge from experience

## Maintenance
Keep policies in sync with `~/.ai-dev-protocol/` (global source).
